# Quick Task 260521-k8b — Unblock Web Build

## Objective

Close the BLOCKER flagged in `.planning/phases/02-medication-catalog/02-VERIFICATION.md` anti-patterns table: `pnpm --filter @meditrack/web build` exiting with `TS2307: Cannot find module '@radix-ui/react-select'`, plus a parallel blocker discovered while triaging it — `docker compose up --build` failing because the web image never builds `@meditrack/shared` before running the SPA build.

## Tasks

### Task 1: Run pnpm install on the host

- **Action:** `pnpm install` from repo root.
- **Why:** The lockfile already pins `@radix-ui/react-select`; the host's `node_modules` was stale. No committed file changes from this step.
- **Verify:** `pnpm --filter @meditrack/web build` exits 0.
- **Done:** Build green.

### Task 2: Add shared-build step to apps/web/Dockerfile

- **Files:** `apps/web/Dockerfile`
- **Action:** Insert `RUN pnpm --filter @meditrack/shared run build` between the `COPY packages/shared` line and the existing `pnpm --filter @meditrack/web run build` step.
- **Why:** `@meditrack/shared`'s `package.json#exports.import` points at `./dist/index.js`. The repo's `.dockerignore` excludes `**/dist`, so the host's prebuilt dist never reaches the image. Without an explicit in-image build, vite's commonjs-resolver fails with `Failed to resolve entry for package @meditrack/shared`. This makes `docker compose up --build` fail on a fresh clone — directly contradicting the README's "golden command" promise (brief §3.3).
- **Verify:** Diff matches the change already in the working tree.
- **Done:** Dockerfile committed.
