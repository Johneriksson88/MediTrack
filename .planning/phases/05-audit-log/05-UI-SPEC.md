**Phase 5 additions / exceptions:**

- **Expand-row affordance (≥md table):** `<TableRow>` is the click target; the chevron icon (`ChevronDown` / `ChevronUp`, `h-4 w-4`) lives in the right-edge `Åtgärd`-equivalent column. Row's intrinsic height (`px-4 py-3` cell padding × 14px text) already exceeds 44 px — no extra geometry needed.
- **Expand-card affordance (<md card):** Whole card is the click target. Tap-to-expand inflates the card vertically; min-height of the card itself (`p-4` × 3 lines content) already exceeds 44 px.
- **AuditDiffPanel inset:** When expanded inside a table row, the panel renders as a single full-width cell (`colSpan={N}`) with `bg-muted/30 px-4 py-4`. Visually inset by the surrounding row borders. Card-mode: panel renders inline inside the card, separated by `<hr className="my-3 border-border" />`.
- **Mobile filter strip (<md):** Tailwind `overflow-x-auto` wrapper, `flex-nowrap`, each combobox trigger uses `flex-shrink-0`. Same precedent as Phase 4 D-82 status-tab strip.
- **"Läs in fler" pagination button:** `min-h-[44px]` touch target, centered below the list, `mt-6 mb-12`. On mobile the page already has `pb-[calc(56px+env(safe-area-inset-bottom))]` to clear the bottom tab bar — no extra padding needed on the button itself.
- **Permalink button (inside diff panel):** `Button variant="ghost" size="sm"` with leading `Link2` icon (`h-3.5 w-3.5 mr-1`). Touch target softened with `h-9` (36 px) — acceptable because this is a desktop-leaning forensic affordance, not a primary mobile action, and lives inside an already-tapped expanded panel where input is intentional. (Recommended deviation; see §Accessibility Floor.)
- **No FAB.** Phase 5 has no create-action — admins only read.

### Destructive Reserved For (Phase 5 additions)

1. **Action chip for `auth.login_failed`** — `bg-destructive/10 text-destructive border border-destructive/30`. The only destructive-tinted chip in the action palette. Signals "this is a security-relevant negative event" without consuming a strong destructive surface.

### 1. `<AuditPage>` — Page-Level Orchestrator

**File:** `apps/web/src/routes/admin/AuditPage.tsx` (replaces the existing stub)

**Query keys (D-69 convention):**
- Events list: `['audit', 'events', { actorUserId, entityType, action, requestId }]` (filters object stable across pages)
- Filters source: `['audit', 'filters']` (no params; small, cached 60s by BE per CONTEXT.md domain)

**URL params (D-103):**
- `actor` — actorUserId (cuid)
- `entity` — entity type enum value
- `action` — action string
- `requestId` — full UUID (only set via requestId-group chip click; not user-editable in v1)

**Loading state (initial fetch, no filters):**

- Event list area: 8 `<Skeleton>` rows of `h-12` at md+ inside the table host; 4 `<Skeleton>` cards of `h-24` at <md. Same shape as Phase 2/3/4 list-loading.

**Loading state (filter switch / load-more):**
- Tab strip / filter bar remains interactive.
- New page append: existing rows stay visible; bottom button enters `disabled` + `Loader2 animate-spin` + label `Läser in fler händelser...`. No skeleton — keeps the eye anchored on the loaded rows.

**Empty states (two distinct paths per D-105 + Claude's discretion in CONTEXT.md `<decisions>`):**

| Trigger | Heading | Body | Visual |
|---------|---------|------|--------|
| No filters AND zero events ever | `Inga händelser ännu` | `Händelser visas här när någon ändrar något i systemet.` | `<EmptyStateCard icon={ShieldCheck}>` — Phase 1 EmptyStateCard pattern, ShieldCheck icon (carry-forward from the stub it replaces — keeps visual continuity for reviewers who have seen the stub) |

**Heading detection:** The page distinguishes the two by checking whether any of the three filter URL params are set. If all three are absent AND the query returns zero events, the page renders the "no events ever" EmptyStateCard. Otherwise, the inline filtered-empty message renders below the filter bar.

**End-of-list state (D-105 + CONTEXT.md `<specifics>`):**

When the query has loaded at least one page AND `hasNextPage === false`:

```jsx
<p className="text-sm text-muted-foreground text-center py-6 mt-4">
  Inga fler händelser att visa.
</p>
```

Renders in place of the "Läs in fler" button. Stays visible while the admin scrolls back through loaded rows.

**Diff Compact Format (Claude's Discretion resolved per CONTEXT.md):**

- **For UPDATE events** (1 changed key): `{key}: {oldVal} → {newVal}` — e.g. `status: skickad → bekraftad`, `currentStock: 3 → 8`.

- **For CREATE events** (no `before`): `Skapad: {entityType}` plus key signature if interesting — e.g. `Skapad: order` or `Skapad: läkemedel · Paracetamol`.
- **For DELETE events** (no `after`): `Borttagen: {entityType}` plus the entity's name field if available — e.g. `Borttagen: läkemedel · Aspirin`.

Values rendered raw (no quotes around strings; numbers as numbers). Long values truncate at 40 chars with `…` suffix (`{val.length > 40 ? val.slice(0, 39) + '…' : val}`).

### 5. `<AuditDiffPanel>` — Diff Panel (D-104)

**File:** `apps/web/src/routes/admin/AuditDiffPanel.tsx`

The expanded-row / expanded-card panel. Renders a key/old/new triplet table plus contextual chrome.

**Props:**
```ts
{
  event: AuditEventResponse;  // includes before, after, requestId, createdAt
  siblingCount?: number;      // number of events sharing this requestId (computed at query layer or fetched ad-hoc)
}
```

**Panel structure:**

```
┌────────────────────────────────────────────────────────────┐
│  [Del av begäran <last8> · X händelser]   {ISO timestamp}  │
│  ────────────────────────────────────────────────────      │
│  ┌──────────────────────────────────────────────────────┐  │
│  │ Fält       │ Före            │ Efter                 │  │
│  ├────────────┼─────────────────┼─────────────────────  │  │
│  │ status     │ skickad         │ bekraftad             │  │
│  │ updatedAt  │ 2026-05-22T...  │ 2026-05-22T...        │  │
│  └──────────────────────────────────────────────────────┘  │
│                                          [Kopiera permalink] │
└────────────────────────────────────────────────────────────┘
```

**Panel header (top-right):** The full ISO timestamp from `createdAt`, `text-xs font-mono text-muted-foreground`. Always visible — supplements the tooltip pattern for accessibility (the tooltip is hover-only on desktop; the panel ensures the ISO is also reachable by tab + read).

**Panel header (top-left):** The `<RequestIdGroupChip>` if `siblingCount > 1` OR `requestId` is set. Label: `Del av begäran {requestId.slice(-8)} · {siblingCount} händelser`. Single-event requests (siblingCount === 1) suppress the chip (no group to navigate to). The chip is a `<Link>` to `/admin/audit?requestId={requestId}` (full UUID).

**Diff table (D-104):**

shadcn `<Table>` rendered inside the panel, sized to content (`w-full`).

| Column | Width | Cell content |
|--------|-------|--------------|
| `Fält` | `w-[180px]` | Field key — `text-sm font-semibold text-foreground` |
| `Före` | `min-w-[200px]` | Old value — see §Value Rendering |
| `Efter` | `min-w-[200px]` | New value — see §Value Rendering |

**Column header style:** `text-xs font-semibold text-muted-foreground uppercase tracking-wide`. `bg-transparent` (no muted bg — keeps the panel visually distinct from the parent table headers).

**Row computation (D-95):**

The panel computes diffs at render time given `before` and `after` JSON objects:

- **CREATE** (`before === null && after !== null`): Render every key from `after`. The `Före` column shows `—` (em dash, `text-muted-foreground`). The `Efter` column shows the value.
- **DELETE** (`before !== null && after === null`): Render every key from `before`. `Före` shows the value, `Efter` shows `—`.
- **UPDATE** (both non-null): Render only keys where `before[key] !== after[key]` (deep equality via `JSON.stringify` per key — cheap; allowlisted columns keep the JSONs small). If zero keys differ (theoretical edge case — should not happen in practice since the audit hook only fires on a real mutation), render a single row: `Fält: (ingen ändring) · Före: — · Efter: —`.

**Value Rendering:**

Values are typed against the allowlisted columns (D-97). Per-type rendering rules:

| Value type | Render as | Example |
|------------|-----------|---------|
| `string` | `text-sm text-foreground font-mono whitespace-pre-wrap break-all max-w-[280px]` | `bekraftad` |
| `number` | `text-sm text-foreground font-mono` | `8` |
| `boolean` | `text-sm text-foreground font-mono` | `true` / `false` |
| `null` | `text-sm text-muted-foreground italic` | `null` |
| `Date` (ISO string) | `text-sm font-mono text-foreground` | `2026-05-22T14:02:17.123Z` |
| Object/Array | `<pre className="text-xs font-mono whitespace-pre-wrap break-all max-w-[280px]">{JSON.stringify(value, null, 2)}</pre>` | nested JSON |

**`font-mono`:** Used inside the diff table for values to give them a deliberately code-like feel — this is a forensic surface; values should read as data, not prose. The mono font is the system default (`ui-monospace, …`) — no webfont. This is an addition to Phase 5 typography reserved exclusively for value cells inside AuditDiffPanel; it does NOT cascade to any other surface.

**Panel footer (bottom-right):** `<Button variant="ghost" size="sm" onClick={copyPermalink}>` with leading `<Link2 className="h-3.5 w-3.5 mr-1" />` and label `Kopiera permalink`. Right-aligned via `flex justify-end mt-3`.

**Copy permalink interaction (Claude's Discretion resolved — placement = panel bottom-right):**

1. Click → compute the canonical filtered URL for this single event:
   - Base: `${window.location.origin}/admin/audit`
   - Params: include only the filters that would produce a list containing this event (e.g. `?actor={event.actorUserId}&action={event.action}&requestId={event.requestId}`). Building from the existing event guarantees the URL re-fetches a page containing this event.
2. `navigator.clipboard.writeText(url)` → on success: `toast.success('Permalink kopierad.')` (D-104, locked copy).
3. On failure (clipboard API unavailable / permission denied): `toast.error('Kunde inte kopiera permalink.')`.
4. Button does not change state visually (no spinner) — clipboard write is synchronous from user perspective.

**Panel layout (≥md inside table):** Lives inside a `<TableCell colSpan={6}>` with `bg-muted/30 px-4 py-4 border-b border-border`. Width = full table width.

**Panel layout (<md inside card):** Lives inside the card's content area, below the diff-summary row, separated by `<hr className="my-3 border-border" />`. No `bg-muted/30` background — the card already has `bg-card` and the contrast against the parent card border is sufficient.

**Accessibility:**

- `id={diffPanelId}` matches the row's `aria-controls`.
- Panel is `<div role="region" aria-label="Detaljer för händelse">`.
- Permalink button has inherent label from its text; no extra `aria-label` needed.
- RequestId-group chip uses `<a>` (semantic link) — screen reader announces as navigation.

## Responsive Breakpoint Contracts

| Breakpoint | Filter bar | Event list | AuditDiffPanel host |
|-----------|------------|-----------|---------------------|
| <768px (mobile) | Horizontally-scrollable strip (`overflow-x-auto`, edge-to-edge); clear-all trailing the comboboxes inside the strip | Card list (`<AuditCardList>` — `block md:hidden`) | Inline below the diff-summary row inside the card, separated by `<hr>` |
| ≥768px (md) | Horizontal row, `flex flex-wrap items-center gap-2`; clear-all right-aligned via `ml-auto` | Table (`<AuditTable>` — `hidden md:block`) | Second `<TableRow>` immediately below the parent row, `<TableCell colSpan={6}>` host |
| ≥1024px (lg) | Same as md, more horizontal breathing room (page padding `p-8`) | Same as md | Same as md |
| ≥1440px (xl) | Same as md | Same as md | Same as md |

**No horizontal scroll at any breakpoint** (UX-01). Mobile filter strip uses `overflow-x-auto` deliberately and scopes the horizontal-scroll to that single element (not the whole page). Table at md+ uses `overflow-x-auto` wrapper as defensive fallback; columns are sized to fit at 768px with `min-w-0` on flex children.

**Bottom tab bar awareness (mobile):** Page main-content area already has `pb-[calc(56px+env(safe-area-inset-bottom))]` from Phase 1's shell — the audit page does not need to repeat this. The `mb-12` on the bottom pagination zone gives an extra ~48 px of breathing room above the safe-area-inset zone.

**Combobox popover at mobile (<md):** Radix-native portal renders the popover at the viewport level; positioning is automatic. Width: `w-[280px]` for Användare/Åtgärd, `w-[240px]` for Entitetstyp. On a 360px viewport with 16px page padding (32px total), the popover comfortably fits (max 280px = 280 vs 328px available).

Carry-forward from Phases 1–4. Phase 5 additions:

## No-Horizontal-Scroll Guarantee

Verified requirements per UX-01:

| Breakpoint | Guarantee | Implementation |
|-----------|-----------|----------------|
| 360px | Filter bar uses `overflow-x-auto` scoped to itself; main content `w-full overflow-x-hidden`. Cards `w-full` with `p-4` inset. Combobox popover (max 280 px) fits within viewport. | `flex-nowrap overflow-x-auto` on filter strip, `min-w-0` on all flex children |
| 768px | Table `overflow-x-auto` wrapper as defensive fallback; columns sized to fit ≥768. Sidebar 64px icon-only; main content `flex-1 min-w-0`. | Column widths sum to ~860 px content + `min-w-0` parent |
| 1024px | Sidebar 240px; main content fills remainder; container `max-w-7xl mx-auto px-8` inside. | Same as Phase 1 layout |
| 1440px | Same as 1024px. AuditDiffPanel host cell expands to full table width. | No change |

**Rule:** Long string values in AuditDiffPanel use `whitespace-pre-wrap break-all max-w-[280px]` so a giant JSON blob cannot blow out a parent flex/table column. Long Användare names truncate via `truncate max-w-[180px]` on the trigger label only — the popover row shows full name + email.
