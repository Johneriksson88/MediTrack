`listAuditEvents(filters: AuditEventListQuery): Promise<AuditEventListResponse>`:
1. Decode `filters.cursor` if present (base64 → JSON → `{createdAt, id}`). On invalid cursor, throw `ValidationFailedError` (existing class — reuse the Phase 4 error catalog per D-19 carry-forward).
2. Build the Prisma `where` clause:
   - If `filters.actorUserId` → `actorUserId: filters.actorUserId`.
   - If `filters.entityType` → `entityType: filters.entityType`.
   - If `filters.action` → `action: filters.action`.
   - If `filters.requestId` → `requestId: filters.requestId`.
   - Cursor: `OR: [{ createdAt: { lt: cursorAt } }, { AND: [{ createdAt: cursorAt }, { id: { lt: cursorId } }] }]` (deterministic tiebreak).
3. `prisma.auditEvent.findMany({ where, orderBy: [{ createdAt: 'desc' }, { id: 'desc' }], take: limit + 1 })`.
4. If results.length > limit, `hasMore = true`; drop the +1; encode `nextCursor` from the now-last row's `(createdAt, id)`; else `nextCursor = null`.
5. For the actor JOIN: collect distinct `actorUserId`s from the returned rows; `prisma.user.findMany({ where: { id: { in: ids } }, select: { id, name, email, role } })`; build a `Map<id, actor>`; attach `actor: map.get(row.actorUserId) ?? null` to each event response (the `actorUserId` field stays in the row for compatibility).
6. Map rows → `auditEventResponse` shape (ensure `createdAt` is serialized as ISO string — Prisma returns Date; call `.toISOString()` or rely on JSON.stringify default, but the Zod response schema expects `z.string().datetime()` so explicit `.toISOString()` is safer).

`listAuditFilters(): Promise<AuditFiltersResponse>`:
1. Module-scope memo: `let cache: { data: AuditFiltersResponse; expiresAt: number } | null = null` and a TTL constant `FILTERS_CACHE_MS = 60_000`. If `cache && cache.expiresAt > Date.now()` → return `cache.data`.

   - `prisma.auditEvent.groupBy({ by: ['actorUserId'], where: { actorUserId: { not: null } } })` → collect ids → `prisma.user.findMany({ where: { id: { in: ids } }, select: { id, name, email } })` → sort by name.
   - `prisma.auditEvent.groupBy({ by: ['entityType'] })` → sort alphabetically.
   - `prisma.auditEvent.groupBy({ by: ['action'] })` → sort alphabetically.
3. Build and cache the response. Return it.
