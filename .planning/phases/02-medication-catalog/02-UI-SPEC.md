## Spacing Scale

Carry-forward from Phase 1 — no changes. All layout values multiples of 4px.

| Token | px | Tailwind class | Primary usage |
|-------|----|----------------|---------------|
| 1 | 4px | `p-1` / `gap-1` | Icon-label gap |
| 2 | 8px | `p-2` / `gap-2` | Badge inner padding, compact spacing |
| 3 | 12px | `p-3` | Icon button padding at md |
| 4 | 16px | `p-4` / `gap-4` | Card padding, form field gaps, page padding mobile |
| 5 | 20px | `p-5` | Internal vertical padding step |
| 6 | 24px | `p-6` / `gap-6` | Page padding at md, section spacing |
| 8 | 32px | `p-8` | Page padding at lg+ |
| 12 | 48px | `py-12` | Vertical breathing room |

**Phase 2 additions / exceptions:**

- Sheet footer: `p-4` with `border-t border-border`. On mobile, add `pb-[calc(1rem+56px+env(safe-area-inset-bottom))]` to clear the 56px bottom tab bar and iOS home indicator.
- Inline-edit threshold input: occupies exactly the same bounding box as the number it replaces — no layout shift. Use `w-20` (`80px`) for the `<Input type="number">` replacing the threshold display.
- Table cell padding: `px-4 py-3` (12px vertical, 16px horizontal). Compact but touch-safe because the Åtgärd cell contains the row-click affordance.
- FAB (mobile): `bottom-[calc(56px+env(safe-area-inset-bottom)+16px)]` `right-4` — clears tab bar + 16px gap.
- Filter chip row: `gap-2` between chips/inputs, `py-3` vertical padding on the row container.
- Count banner: `px-4 py-2` (8px vertical), `mb-4` before the table/card list.

### 5. `<InlineEditThreshold>` — Inline-Edit Threshold

**File:** `apps/web/src/components/InlineEditThreshold.tsx`

Props: `{ value: number; careUnitMedicationId: string; onSave: (newValue: number) => void }`

**Visual states:**

| State | Visual | Tailwind classes |
|-------|--------|-----------------|
| Idle | Number with hover underline affordance | `text-sm text-foreground underline decoration-dotted cursor-pointer select-none` |
| Editing | `<Input type="number">` replacing the number | `w-20 h-8 text-sm px-2 py-1 focus-visible:ring-2 focus-visible:ring-primary focus-visible:ring-offset-0` |
| Saving | Input disabled, `opacity-60` | `opacity-60 cursor-not-allowed` |
| Error | Input with red border + toast | `border-destructive focus-visible:ring-destructive` |

**Interaction contract:**
1. Click/tap on idle number → enter editing state. Autofocus input. Select-all content.
2. `Enter` or `blur` → trigger save. Transition to saving state. Call `useMutation` (PATCH).
3. `Escape` → cancel. Revert to idle state without saving. No toast.
4. On `onMutate`: optimistically update TanStack Query cache (`['medications', filters]`). Transition to saving state.
5. On success: cache update already applied. Transition to idle state. No toast on success (silent — the number updated in-place is the feedback).
6. On `onError`: rollback cache to previous value. Transition to error state (red border). Show `sonner` toast: `Kunde inte spara — försök igen.` Input remains in editing state (user can retry or press Escape).

**Keyboard:** `e.stopPropagation()` on the `<Input>` keydown to prevent row-click firing. `e.preventDefault()` on Enter to prevent form submission if nested in a form.

**ARIA:** `aria-label="Redigera tröskel för {medicationName}"` on the idle number span. Input: `aria-label="Tröskel"`.

### 6. `<MedicationSheet>` — Create / Edit / View Sheet

**File:** `apps/web/src/routes/lakemedel/MedicationSheet.tsx`

Props: `{ mode: 'create' | 'edit' | 'view'; careUnitMedication?: CareUnitMedicationWithMedication; open: boolean; onOpenChange: (open: boolean) => void }`

**Primitive:** shadcn `<Sheet>`. Direction: `side="right"` on `md`+. On mobile (<md): use `side="bottom"` (vaul-based Sheet variant). The vaul bottom-sheet renders better than `side="right"` at 360px because it gives full viewport width.

**Width (desktop):** `w-[480px]` or `sm:max-w-xl` — standard Sheet right-panel width. Does not change URL.

**Bottom-sheet (mobile):** Full-width, max-height `90dvh`, rounded top corners `rounded-t-2xl`. Scrollable content area.

**Sheet structure:**

```
<SheetHeader>
  <SheetTitle>{heading}</SheetTitle>         ← text-xl font-semibold
  [<NplBadge>Från NPL</NplBadge>]            ← visible in edit mode for npl-source meds
</SheetHeader>
<SheetContent className="overflow-y-auto flex-1">
  {form content — see per-mode below}
</SheetContent>
<SheetFooter className="border-t border-border p-4 flex items-center justify-between gap-2
                        pb-[calc(1rem+56px+env(safe-area-inset-bottom))]">
  {footer actions — see per-mode below}
</SheetFooter>
```

**Headings per mode:**

| mode | SheetTitle | Notes |
|------|-----------|-------|
| `create` | `Lägg till läkemedel` | Typeahead is the first field |
| `edit` | `{medication.name}` | Truncate with `truncate max-w-[360px]` |
| `view` | `{medication.name} · Visning` | Read-only; all inputs disabled or replaced with text |

**Footer per mode:**

| mode | Left | Right |
|------|------|-------|
| `create` | — | `Avbryt` (ghost) + `Spara` (default/blue) |
| `edit` | `Ta bort` (destructive) | `Avbryt` (ghost) + `Spara` (default/blue) |
| `view` | — | `Stäng` (secondary) |

`Spara` is `disabled` while form is submitting (`isSubmitting`). Show `Loader2 animate-spin` inside button when saving.

**Focus management:**
- `create` mode: autofocus the typeahead input when Sheet opens.
- `edit` mode: autofocus the first editable input (Lager field for NPL meds; Namn field for user-created meds).
- `view` mode: autofocus `Stäng` button (only interactive element).
- `Esc` closes Sheet (Radix/shadcn handles this natively).
- `Cmd+Enter` / `Ctrl+Enter` saves (add `keydown` listener on the Sheet content element for `(e.metaKey || e.ctrlKey) && e.key === 'Enter'`).

#### 6a. Create Mode — Typeahead + "Skapa nytt"

**Phase 1:** Typeahead `<Input>` at top. `GET /api/medications/search?q={value}&limit=20`. Query key: `['medication-search', q]`. Debounce: **150ms** (hotter than main search).

**Typeahead dropdown:**
- Renders below the input as a shadcn `<Command>` popover or custom `<ul>` with `role="listbox"`.
- Each result: `{name}` — `{atcCode}` — `{form}` — `{strength}` in one row.
- Max height: `max-h-[240px]` with scroll.
- Loading state: `Söker…` with `Loader2` icon, `text-sm text-muted-foreground`.
- Empty result row: `Inget läkemedel matchade.` + button `Skapa nytt läkemedel` — `text-sm text-muted-foreground` + `Button variant="link" className="ml-1 p-0 h-auto"`.

**On typeahead result select:** Auto-prefill read-only fields:
- `ATC-kod`: read-only text label + `<NplBadge>Från NPL</NplBadge>` beside it.
- `Form`: read-only text label.
- `Styrka`: read-only text label (or empty if null).
- Editable fields: `Lager` (number input, required, min 0) + `Tröskel` (number input, auto-populated from `medicationDefaults.ts` heuristic based on form).
- Info note below prefilled fields: `Från NPL · namn / form / styrka är låsta` in `text-xs text-muted-foreground`.

**"Skapa nytt läkemedel" expanded form** (triggered by empty-result button or manually):
Expands below the typeahead with:
- `Namn` — text input, required
- `ATC-kod` — text input, required
- `Form` — select dropdown, required (uses top-N from `medicationForms.ts`)
- `Styrka` — text input, optional
- `Lager` — number input, required, min 0
- `Tröskel` — number input, auto-populated from form selection, editable

**Form validation:** react-hook-form + `@hookform/resolvers/zod`. Schema from `packages/shared/src/contracts/medication.ts`. Inline field errors: `text-xs text-destructive mt-1` below the field.

#### 6b. Edit Mode — RBAC-Aware Fields

**For `Medication.source = 'npl'`:**
- `Namn`, `ATC-kod`, `Form`, `Styrka`: rendered as read-only `<p>` labels, not inputs. `text-sm text-foreground`.
- Lock note: `<NplBadge>Från NPL · namn / form / styrka är låsta</NplBadge>` displayed prominently below the SheetTitle.
- `Lager`: number input, editable. Required, min 0.
- `Tröskel`: number input, editable. Required, min 1.

**For `Medication.source = 'user'`:**
- All fields editable: `Namn`, `ATC-kod`, `Form` (select), `Styrka`, `Lager`, `Tröskel`.

#### 6c. View Mode (sjuksköterska)

All fields rendered as read-only labels, same layout as edit mode but no inputs. Footer has only `Stäng`. No `Ta bort`. No `Spara`. `<SheetTitle>` appends `· Visning` suffix. Entire form `aria-disabled` or fields `disabled` (use `<fieldset disabled>` wrapping the form for clean disabled propagation).

- When inactive: `variant="outline"`, default slate border.

## Accessibility Floor

Additions to Phase 1 baseline (Phase 1 requirements carry forward unchanged).

| Requirement | Implementation |
|-------------|----------------|
| Table keyboard nav | `<TableRow>` has `tabIndex={0}` + `onKeyDown` for `Enter`/`Space` to open Sheet; `role="row"` with `aria-label` |
| Inline-edit ARIA | Idle span: `role="button"` + `aria-label="Redigera tröskel för {name}"`. Input: `aria-label="Tröskel"` |
| Sheet title | `<SheetTitle>` always present — required by Radix Dialog for screen readers |
| AlertDialog focus | Cancel button receives focus on open (shadcn default) — do not override |
| Low-stock banner | `role="alert"` so screen readers announce on mount |
| Low-stock badge | Icon `aria-hidden="true"`; text `Lågt lager` provides the accessible label |
| FAB | `aria-label="Lägg till läkemedel"` — no visible text label on mobile |
| Filter combobox | `aria-label="Filtrera på ATC-kod"` on the trigger |
| Form select | `<label>` associated via `htmlFor` or `<FormLabel>` from react-hook-form / shadcn |
| Filter chip toggle | `aria-pressed={belowThreshold}` on the button |
| Empty state CTA | `<button>` or `<Button>` with clear text label — reachable by keyboard |
| `Visa endast under tröskel` chip | `aria-pressed` reflects active state |
