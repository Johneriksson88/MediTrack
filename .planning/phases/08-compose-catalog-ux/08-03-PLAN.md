<objective>
Ship ORD-08 end-to-end: surface 10 pre-search suggestions on the compose-order MedicationPickerSheet — top-5 most-ordered CareUnitMedications (all-time, per vårdenhet) above top-5 below-threshold rows, deduplicated server-side so the same medication never appears twice. New BE endpoint + shared contract + new shared component + new TanStack hook + one mutation invalidation line + Sheet wiring + integration + component tests. Per D-138 ("Reuses `dashboard.service.listLowStockForUnit` for the Lågt lager half — no duplicate query logic."), the Lågt lager half is sourced exclusively from the existing dashboard service — Option A widens that service + its shared contract in lockstep so the reuse delivers the full PickerSuggestion row shape.

Purpose: Close the third Phase 8 pain point — today's MedicationPickerSheet starts as a blank canvas with "type something to see anything", but the user nearly always wants a low-stock or recently-ordered medication. Surfacing those 10 rows pre-search gives the nurse a single-click path to the most common cases while preserving the existing typeahead for the long tail.
Output: Two new BE files (route + service additions) + two shared contract additions (`pickerSuggestion`/`pickerSuggestionsResponse` in order.ts, widened `lowStockItem` in dashboard.ts) + a widened `listLowStockForUnit` SELECT + adjusted existing dashboard integration tests + two new FE files (block component + hook) + one FE Sheet edit + one FE mutation invalidation line + one BE integration test + one FE component test.
</objective>

    SELECT cum."id" AS "careUnitMedicationId",

           m."name",
           m."atcCode",
           m."form",
           m."strength",
           cum."currentStock",
           cum."lowStockThreshold",
           COUNT(ol."id") AS "orderCount"
    FROM "CareUnitMedication" cum
    JOIN "Medication" m ON cum."medicationId" = m."id"
