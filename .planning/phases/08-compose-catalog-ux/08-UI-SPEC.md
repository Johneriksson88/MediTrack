## Token-Class Hard Rule (Phase 8 specific)

Per the Phase 7 UI review (`.planning/ui-reviews/07-UI-REVIEW.md` — 30 hardcoded hex literals across
shell components flagged as architectural debt), **every color in this UI-SPEC is expressed as a
semantic Tailwind token class** that resolves through the CSS variables in `apps/web/src/index.css`:

- `bg-background`, `bg-card`, `bg-muted`, `bg-muted/50`, `bg-popover`
- `text-foreground`, `text-muted-foreground`, `text-primary`, `text-destructive`, `text-destructive-foreground`
- `border-border`, `border-input`, `border-destructive`, `border-destructive/30`
- `ring-primary`, `ring-ring`, `bg-destructive/10`, `bg-primary/10`

**Hex literals are forbidden in Phase 8 component code.** If a hue is needed that no token covers
(e.g. ConfidenceBadge's green/yellow/slate set from Phase 6), reuse the existing component
(`<ConfidenceBadge>`, `<LowStockBadge>`) rather than re-declaring its hex inline. Phase 8 introduces
no such new hue; every Phase 8 surface lives entirely within the seven token families above.

The "Reference hex" column in the Color table below is documentation only — the implementation
binds to the CSS-var class. Phase 8 must not contribute one additional hex literal to the codebase.

## Components & States

### 1. `AtcCodeCombobox` (`apps/web/src/components/AtcCodeCombobox.tsx`) — NEW

Shared Popover+Command combobox for the global ATC-code list. Mirrors the `TherapeuticClassCombobox`
structure verbatim with one functional difference: ATC accepts free-text fallback for codes not in
the served list (so a freshly-added user-created medication can carry a brand-new code).

**Props interface (locked):**

```ts
interface AtcCodeComboboxProps {
  /** Current value. Empty string '' = unset (matches LakemedelFilter URL-state convention). */
  value: string;
  /** Called when the user picks a code, types a free-text fallback, or clears the field. */
  onChange: (next: string) => void;
  /** Trigger placeholder when value is empty. Default: 'ATC-kod ▾'. */
  placeholder?: string;
  /** CommandInput placeholder. Default: 'Sök ATC-kod…'. */
  searchPlaceholder?: string;
  /** aria-label on the trigger button. Default: 'Välj ATC-kod'. */
  ariaLabel?: string;
  /** Consumer-controlled trigger sizing. */
  triggerClassName?: string;
  /** Show X clear affordance when value is set. Default: true. */
  clearable?: boolean;
  /** Disable the trigger (e.g. while a parent mutation is pending). Default: false. */
  disabled?: boolean;
}
```

**Trigger structure (mirrors TherapeuticClassCombobox lines 82-122):**

```
<Popover open={open} onOpenChange={setOpen}>
  <PopoverTrigger asChild>
    <Button
      variant="outline"
      role="combobox"
      aria-expanded={open}
      aria-label={ariaLabel}
      className={cn('justify-between', triggerClassName)}
      type="button"
      disabled={disabled}
    >
      <span className="truncate">{value || placeholder}</span>
      {value && clearable ? (
        <span                                       ← clear affordance, span not nested button
          role="button"
          tabIndex={0}
          aria-label="Rensa ATC-kod"
          className="ml-2 inline-flex h-4 w-4 items-center justify-center rounded-sm opacity-70 hover:opacity-100 focus:outline-none focus:ring-1 focus:ring-ring"
          onClick={(e) => { e.stopPropagation(); onChange(''); }}
          onKeyDown={...}
        >
          <X className="h-4 w-4" />
        </span>
      ) : (
        <ChevronsUpDown className="ml-2 h-4 w-4 shrink-0 opacity-50" />
      )}
    </Button>
  </PopoverTrigger>
  <PopoverContent
    align="start"
    className="w-[--radix-popover-trigger-width] min-w-[240px] p-0"
  >
    <Command>
      <CommandInput
        placeholder={searchPlaceholder}
        value={query}
        onValueChange={setQuery}
      />
      <CommandList>
        {codesQuery.isLoading && (
          <div className="p-3 text-sm text-muted-foreground flex items-center gap-2">
            <Loader2 className="h-4 w-4 animate-spin" aria-hidden="true" />
            Laddar ATC-koder…
          </div>
        )}
        {!codesQuery.isLoading && (
          <>
            <CommandEmpty>Inga träffar</CommandEmpty>
            <CommandGroup>
              {filteredCodes.map((code) => (
                <CommandItem
                  key={code}
                  value={code}
                  onSelect={() => { onChange(code); setOpen(false); }}
                >
                  <Check className={cn('mr-2 h-4 w-4', value === code ? 'opacity-100' : 'opacity-0')} />
                  <span className="text-xs font-semibold tabular-nums">{code}</span>
                </CommandItem>
              ))}
              {/* Free-text fallback row — only if typed value is not in served list */}
              {query.trim() && !filteredCodes.includes(query.trim().toUpperCase()) && (
                <CommandItem
                  key={`__free__${query}`}
                  value={`__free__${query}`}
                  onSelect={() => { onChange(query.trim().toUpperCase()); setOpen(false); }}
                >
                  <Check className="mr-2 h-4 w-4 opacity-0" />
                  <span className="text-xs font-semibold tabular-nums">
                    {query.trim().toUpperCase()}
                  </span>
                  <span className="text-xs text-muted-foreground ml-2">(fri sökning)</span>
                </CommandItem>
              )}
            </CommandGroup>
          </>
        )}
      </CommandList>
    </Command>
  </PopoverContent>
</Popover>
```

**Data source:** `useAtcCodesQuery()` (new hook, see §Data Caching). Query key `['atc-codes']`,
`staleTime: Infinity`, `refetchOnWindowFocus: false`. The cmdk library's built-in matcher does
all client-side filtering against `filteredCodes` — typing "N0" narrows visible rows; no debounce.

**States:**

| State | Visual |
|-------|--------|
| Idle, empty value | Trigger renders placeholder (`ATC-kod ▾` in LakemedelFilter; `Välj ATC-kod` in MedicationSheet) + `ChevronsUpDown` icon |
| Idle, value set | Trigger renders the code value (e.g. `N02BE01`) + clear `X` icon (when `clearable`) |
| Open, codes loading | Dropdown shows `Loader2` + `Laddar ATC-koder…` text — only on the very first request before `staleTime: Infinity` kicks in. After any user has opened the combobox once in a session, this state is rare. |
| Open, codes loaded, no query | Full list rendered, scrollable via cmdk's `max-h-[300px]` default |
| Open, codes loaded, typing | cmdk filters the list live; `Check` mark on the currently-selected row, opacity-0 on all others |
| Open, codes loaded, typed value not in list | Above the filtered list, the free-text fallback row appears: `{TYPED} (fri sökning)` |
| Open, codes loaded, zero matches AND no free-text candidate | `<CommandEmpty>Inga träffar</CommandEmpty>` |
| Trigger focus (keyboard) | `focus-visible:ring-2 focus-visible:ring-primary focus-visible:ring-offset-2` (inherited from shadcn Button) |
| Disabled | `opacity-50 cursor-not-allowed` (shadcn Button default disabled styling) |

**Accessibility:**

- Trigger: `role="combobox"`, `aria-expanded={open}`, `aria-label={ariaLabel}`.
- Clear affordance: `role="button" tabIndex={0} aria-label="Rensa ATC-kod"`. Keyboard
  Enter/Space activates it. `stopPropagation` to avoid opening the popover when clearing.
- `CommandList` rows: keyboard arrow nav handled by cmdk; `Enter` selects.
- `Esc` closes (Radix native).

**Touch target:** Trigger `h-10` (40px) — below the 44px nominal floor but acceptable per the
Phase 6 precedent for in-form combobox triggers (Slutgiltig klass field, AI block "Använd förslag"
documented as `h-9` deviation). The clear `X` is `h-4 w-4` inside an `inline-flex` slot — the
**entire trigger button** (40px tall) is the touch target; the X is hit-tested via stopPropagation
inside the same Button.

export function useAtcCodesQuery() {

## Interaction Contracts

### AtcCodeCombobox flow

```
[idle, empty value]
  → user clicks trigger
  → Popover opens; codes load (or are already cached at staleTime: Infinity)
  → user types into CommandInput
  → cmdk filters the list live; visible rows narrow
  → if typed value matches an existing code:
      → user clicks the matching row → onChange(code), popover closes
  → if typed value does not match any code:
      → free-text fallback row `{TYPED} (fri sökning)` appears at end of list
      → user clicks free-text row → onChange(TYPED.toUpperCase()), popover closes
  → if typed value is empty and no codes loaded yet (first ever open):
      → `Laddar ATC-koder…` placeholder shows for the network round-trip

[idle, value set]
  → trigger shows the code + X clear icon
  → user clicks X → onChange('') → trigger reverts to placeholder
  → user clicks trigger body (not X) → popover opens with the value-row marked Check
  → user types → cmdk filters from full list, the previously-selected row may or may not be visible
```

### Picker Suggestions block flow

```
[picker opens, searchQuery === '']
  → MedicationPickerSheet renders the search input + PickerSuggestionsBlock
  → block fires usePickerSuggestionsQuery(orderId)
  → initial skeleton renders (3 row skeletons per section)
  → data arrives → sections render with sticky headers
  → user clicks a suggestion row → onRowClick(careUnitMedicationId)
    → identical to existing handleRowClick:
      → await addLineMutation.mutateAsync({ orderId, careUnitMedicationId, quantity: 1 })
      → on success: Sheet closes, searchQuery resets to ''
      → on error: Sheet stays open, existing toast fires

[user starts typing]
  → debouncedQ flips from '' to non-empty after 150ms
  → PickerSuggestionsBlock unmounts (conditional render gate)
  → existing search-results branches take over (Söker… → results → CAT-10 empty state)
  → no flicker: the suggestions block's exit and the search-results' enter are unconditional
    by the same `debouncedQ === ''` predicate

[user clears the search input back to empty]
  → debouncedQ flips back to '' after 150ms
  → PickerSuggestionsBlock remounts
  → query cache hit (staleTime 30s) → instant render, no skeleton
```

### CAT-10 empty-state flow

```
[user opens MedicationSheet in create mode]
  → typeahead input autofocused, empty
  → user types "paracetamol" → searchQuery fires after 150ms debounce (Phase 2 D-44)
  → BE returns { results: [...], globalCatalogMatchCount: N }

Case 1: results.length > 0
  → existing typeahead-result list renders unchanged

Case 2: results.length === 0 AND globalCatalogMatchCount === 0
  → Variant A empty-state: "Inget i NPL matchade »paracetamol«."
  → sub-line with inline "skapa ett nytt läkemedel" link
  → user clicks link → setShowCreateForm(true), user-create form renders

Case 3: results.length === 0 AND globalCatalogMatchCount > 0
  → Variant B empty-state: "Alla träffar finns redan i din vårdenhet."
  → sub-line with inline "skapa ett nytt läkemedel" link (same link as Variant A)
  → user clicks link → setShowCreateForm(true), user-create form renders
```

## Data Caching

Phase 8 introduces two new query keys and updates one mutation invalidation list.

| Query key | Hook | staleTime | refetchOnWindowFocus | Invalidated by |
|-----------|------|-----------|---------------------|----------------|
| `['atc-codes']` | `useAtcCodesQuery` | `Infinity` | `false` | `useCreateMedication.onSuccess` (new line) |
| `['order-picker-suggestions', orderId]` | `usePickerSuggestionsQuery` | `30_000` (30s) | `false` | `useAddOrderLine.onSuccess` (existing — already invalidates `['order', orderId]`; **add** `['order-picker-suggestions', orderId]` so re-opening the picker reflects newly-added lines) |

The picker-suggestions invalidation is conservative — most-ordered and low-stock rankings can
shift on add-line (a med that was the 5th most-ordered may climb if the same nurse adds it again
in a follow-up order, though this is rare in v1's small dataset). 30s staleTime softens the cost:
two consecutive picker opens within 30s reuse the same data; after that the natural staleness
triggers a refetch on next open.

**No mutation outside `useAddOrderLine` invalidates picker-suggestions.** Deliveries don't —
the BE endpoint reads `dashboard.service.listLowStockForUnit` which is invalidated by
`useDeliverOrder.onSuccess` against `['dashboard', 'low-stock']`, but the picker-suggestions
endpoint reads it fresh on each call (no cross-key cache dependency).
