## Phase Goal

**As a** nurse, apotekare, or admin using MediTrack, **I want to** see every order labelled with a unique, human-readable order number that persists through its full lifecycle, **so that** I can reference an order conversationally (in a hand-off, in an audit lookup, in a phone call) without quoting an opaque cuid.

<objective>
Light up Phase 10 ORD-11 in the FE: every order-rendering surface promotes orderNumber to identity-level visual prominence. OrdersTable / DraftsTable get a leftmost Best.nr column; card variants and DraftCard surface orderNumber as the heading; ComposeOrderPage gets a Beställning ORD-YYYY-#### H1; DashboardOrdersCard row layout promotes orderNumber over createdBy.name; SubmitConfirmationBanner copy embeds the number. aria-labels at every row site update to reference orderNumber rather than formatRelative(timestamp). Extended component tests assert the new copy + DOM presence. After this plan, the mobile-first sc04 screenshot harness is re-run; a final human-verify checkpoint confirms no 360 px horizontal-scroll regression and that the new Best.nr column + dashboard heading actually look right on real devices.

Purpose: Phase 10's user-visible deliverable. Plan 01 made the data exist; Plan 02 makes the user able to see it everywhere the roadmap promised (ROADMAP SC#4).
Output: Updated FE components + component tests + re-captured 360 px PNGs + human-verify checkpoint completion notes recorded in 10-02-SUMMARY.md.
</objective>
