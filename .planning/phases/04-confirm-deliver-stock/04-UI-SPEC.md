## Spacing Scale

Carry-forward from Phases 1–3 — no changes. All layout values multiples of 4px.

| Token | px | Tailwind class | Primary usage in Phase 4 |
|-------|----|----------------|--------------------------|
| 1 | 4px | `gap-1` / `p-1` | Icon-label gap in status tabs, actor-trail separator |
| 2 | 8px | `gap-2` / `p-2` | Inline button-to-summary gap in sticky footer |
| 3 | 12px | `px-3` | Status pill horizontal padding (`px-3 py-1`) — carry-forward |
| 4 | 16px | `p-4` / `gap-4` | Page padding (mobile), sticky footer outer padding, banner padding |
| 5 | 20px | `py-5` | Status banner vertical padding (mirrors Phase 3 submit banner) |
| 6 | 24px | `p-6` / `gap-6` | Page padding at md |
| 8 | 32px | `p-8` | Page padding at lg+ |
| 12 | 48px | `py-12` | Empty-state vertical breathing room |

**Phase 4 additions / exceptions:**

- **Action buttons (Mode C/D):** `Bekräfta beställning` and `Markera som levererad` must be `min-h-[44px]` — touch target floor from Phase 1 D-10. At desktop they render right-aligned in the header row; at mobile they render inside the sticky footer (same pattern as D-71).
- **Sticky footer (Mode C/D, mobile):** outer `p-4` + `pb-[calc(1rem+56px+env(safe-area-inset-bottom))]` — verbatim Phase 3 D-71 geometry. The action button is the only content in the Mode C/D footer (no `Kasta`; no summary count); the `Bekräfta`/`Markera levererad` button occupies full width with `w-full`.
- **Status-tab strip (mobile):** Tailwind `overflow-x-auto` wrapper, `flex-nowrap`, tabs rendered as a horizontally scrollable row. Tab minimum width must not compress to < 80px to keep labels legible; use `flex-shrink-0` on each tab trigger.
- **Actor-trail line (Mode D/E):** `text-xs text-muted-foreground`, `flex flex-wrap gap-x-1 gap-y-1`. Dot separators (`·`) are `px-0.5 text-muted-foreground/60`. Wrap gracefully on narrow viewports.

1. **`Bekräfta beställning` button** (Mode C, `<Can action="order:confirm">`) — `Button variant="default"` (blue bg + white text). Primary CTA of Mode C.
2. **`Markera som levererad` button** (Mode D, `<Can action="order:deliver">`) — `Button variant="default"` (blue bg + white text). Primary CTA of Mode D.
3. **Mode D confirmation banner** (`Beställningen är bekräftad — väntar på leverans.`) — `bg-primary/10 border-primary/20 text-primary` tint. Informational, same geometry as Phase 3 submit-confirmation banner.

**Accent must NOT appear on:** Mode E actor-trail line, read-only line cells, status pill backgrounds (each has its own locked color — see `<OrderStatusPill>` below), or the deliver `AlertDialog` confirm button (that is a `Button variant="default"` but styled separately — see Destructive section).

### Destructive Reserved For (Phase 4 additions)

1. **`Markera som levererad` AlertDialog confirm button** (`Markera levererad`) — This is an irreversible, stock-mutating action. The confirm button uses `Button variant="default"` but with a visual warning posture. However, CONTEXT.md D-83 does not prescribe destructive styling for this specific button — the `<AlertDialog>` body copy (`Stocken uppdateras direkt. Detta kan inte ångras.`) is the warning signal. Use `AlertDialogAction` with **default** (not destructive) styling — the action is additive (stock increment), not deletion. Destructive color is reserved for removes and discards per Phase 2/3 precedent.
2. **Toast for 422 `medication_removed`** — `toast.error(...)` with message `{med.name} har tagits bort — återställ läkemedlet i registret innan leverans.` Rendered as a `toast.error` call (red tint) per Phase 2/3 error-toast convention.
3. **Toast for 409 `order_transition_invalid`** — `toast.error(...)` with message `Beställningen har redan {ORDER_STATUS_LABELS[details.from]}.` Also a `toast.error` call.

## Components — Phase 4 Deltas

### 1. `<BestallningarPage>` — Status-Tab Filter (D-82)

**File:** `apps/web/src/routes/bestallningar/BestallningarPage.tsx` (extend)

**Primary visual anchor:** the active tab underline (accent blue) + the first row of the order list table/card.

**New behavior:** Replace the static `?status=utkast` query with a URL-backed status-tab filter. Tab value is read from `useSearchParams()` (`?status=skickad`). Navigating tabs updates the URL without a full page reload.

#### Tab Strip Layout

```
┌──────────────────────────────────────────────────────────┐
│  [ Utkast ] [ Skickade ] [ Bekräftade ] [ Levererade ] [ Alla ] │
│  ─────────────────────────────────────────────────────   │
│  (underline indicator tracks active tab)                 │
├──────────────────────────────────────────────────────────┤
│  Order list (table ≥md / card list <md)                  │
│  Columns vary by active tab — see column spec below      │
└──────────────────────────────────────────────────────────┘
```

**Primitive:** shadcn `<Tabs>` with `value` bound to the URL `status` param. `onValueChange` calls `setSearchParams({ status: newValue })`. The `Alla` tab sets `status=alla` in the URL; the query layer translates to a comma-list of all four statuses.

**Tab definitions (left-to-right):**

| Tab | URL param value | Query status sent to API |
|-----|-----------------|--------------------------|
| `Utkast` | `utkast` (default when param absent) | `?status=utkast` |
| `Skickade` | `skickad` | `?status=skickad` |
| `Bekräftade` | `bekraftad` | `?status=bekraftad` |
| `Levererade` | `levererad` | `?status=levererad` |
| `Alla` | `alla` | `?status=utkast,skickad,bekraftad,levererad` |

**Default:** When no `?status` param is present, default to `utkast` (Phase 3 back-compat — D-82).

**Mobile (`<sm`) tab strip:** The five tabs overflow at 360px. Render the tab strip as `overflow-x-auto` with `flex-nowrap`. Each `<TabsTrigger>` uses `flex-shrink-0` and `whitespace-nowrap`. No horizontal scroll indicator — the visual cutoff of the rightmost tab implies scrollability. No badge counts in v1 (deferred per `<deferred>` section of CONTEXT.md).

**Aesthetic:** Underlined-tabs style — `<TabsList>` renders as a borderless strip with `border-b border-border` on the container and a `border-b-2 border-primary` underline on the active `<TabsTrigger>`. This matches the Phase 2 filter/tab aesthetic. The shadcn `new-york` preset default `<TabsList>` uses a pill/segment appearance — override with `className="bg-transparent border-b border-border rounded-none p-0"` on `<TabsList>` and `className="rounded-none border-b-2 border-transparent data-[state=active]:border-primary data-[state=active]:text-foreground font-semibold text-xs text-muted-foreground pb-2"` on each `<TabsTrigger>`.

### 2. `<ComposeOrderPage>` — Mode Extensions (D-83)

**File:** `apps/web/src/routes/bestallningar/ComposeOrderPage.tsx` (extend)

#### Mode Routing

```
switch (order.status) {
  case 'utkast':   → Mode A (Phase 3, unchanged)
  case 'skickad':
    if viewer is apotekare/admin → Mode C (new, Phase 4)
    else → Mode B (Phase 3, read-only, no action button)

}
```

**`Bekräfta beställning` button:**
- Gate: `<Can action="order:confirm">` (renders only when viewer is apotekare/admin).
- Position desktop (≥md): right-aligned in the header row, flex-row with the page heading and status pill.
- Position mobile (<md): inside the sticky footer, `w-full`, clearing the bottom tab bar per D-71 geometry.
- Variant: `Button variant="default"` (blue).
- Interaction (pessimistic per D-42):
  1. Click → button disabled + `Loader2 animate-spin`.
  2. `POST /api/orders/:id/confirm` fires.
  3. On success: `queryClient.setQueryData(['order', id], response)` + `invalidateQueries(['orders', { status: 'skickad' }])` + `invalidateQueries(['orders', { status: 'bekraftad' }])`. Page re-renders in Mode D. `toast.success('Bekräftad')`.

  5. On 404: `toast.error('Beställning hittades inte.')` + navigate to `/bestallningar`.
  6. On other error: `toast.error('Kunde inte spara — försök igen.')`.

**Read-only line list:** No steppers, no trash buttons, no `Lägg till läkemedel`. Quantities render as static `text-sm font-semibold` — same as Phase 3 Mode B locked rendering. The `OrderLineTable` / `OrderLineCardList` from Phase 3 reuse `isLocked={true}`.

#### Sjuksköterska View — Read-Only Modes (B / D partial / E)

When viewer is sjuksköterska (i.e., `useCan('order:confirm')` and `useCan('order:deliver')` both return `false`):

- **Skickad status:** Renders Phase 3 Mode B (submit-confirmation banner, read-only lines). No `Bekräfta beställning` button. No sticky footer. Unchanged from Phase 3.
- **Bekräftad status:** Read-only line list + Mode D banner (`Beställningen är bekräftad — väntar på leverans.`) + partial actor trail. No `Markera som levererad` button. No sticky footer. The Mode D banner is rendered for all viewers (it communicates status, not role).
- **Levererad status:** Mode E layout unchanged — always the same for all roles (no action button exists regardless of role).

**Empty action area treatment:** No placeholder, no disabled button, no muted "Du saknar behörighet" text. The banner + read-only line list is complete UX; the absence of an action button is self-explanatory in context of the status. This keeps the view clean and avoids exposing role-gate messaging to nurses unnecessarily. `<Can>` renders nothing when the predicate is false — no wrapper div remains.

### 3. `<DeliverConfirmDialog>` — Deliver Confirmation AlertDialog (D-83)

**File:** `apps/web/src/routes/bestallningar/DeliverConfirmDialog.tsx` (new)

Props: `{ open: boolean; onOpenChange: (open: boolean) => void; onConfirm: () => void; isDelivering: boolean }`

Primitive: shadcn `<AlertDialog>`. Parallel shape to Phase 3 `<DiscardDraftDialog>` and Phase 2 `<DeleteMedicationDialog>`.

```
<AlertDialogTitle>Markera som levererad?</AlertDialogTitle>
<AlertDialogDescription>
  Stocken uppdateras direkt. Detta kan inte ångras.
</AlertDialogDescription>
<AlertDialogCancel>Avbryt</AlertDialogCancel>       ← default-focused (shadcn default)
<AlertDialogAction>Markera levererad</AlertDialogAction>
```

- `<AlertDialogCancel>` default-focused. `Escape` cancels (Radix native).
- `<AlertDialogAction>` is NOT styled destructive — the action adds stock. Use `AlertDialogAction` default styling (accent blue). CONTEXT.md does not prescribe destructive here; the body copy is the caution signal.
- When `isDelivering === true`: `<AlertDialogAction>` shows `Loader2 animate-spin`, is `disabled`. `<AlertDialogCancel>` also `disabled` while in-flight.
- `onConfirm` fires `useDeliverOrder` mutation:
  1. `POST /api/orders/:id/deliver`.
  2. On success: `queryClient.setQueryData(['order', id], response)` + `invalidateQueries(['orders', ...])` (all statuses) + `invalidateQueries(['medications', ...])` (Phase 6 NTF-01 hook). Dialog closes. Page re-renders in Mode E. `toast.success('Levererad — lagret uppdaterat')`.
  3. On 409 `order_transition_invalid`: dialog closes. `toast.error('Beställningen har redan {ORDER_STATUS_LABELS[details.from]}.')` + `invalidateQueries(['order', id])`.
  4. On 422 `medication_removed`: dialog closes. `toast.error('{med.name} har tagits bort — återställ läkemedlet i registret innan leverans.')`. Where `med.name` is extracted from the 422 response body `details`.
  5. On other error: dialog stays open. `toast.error('Kunde inte spara — försök igen.')`. Action re-enables.

### 5. Actor Trail — Shared Component

**File:** `apps/web/src/routes/bestallningar/OrderActorTrail.tsx` (new)

Renders the populated actor + timestamp segments as a single formatted line.

Props:
```ts
{
  createdBy: { name: string } | null;
  createdAt: string;
  submittedBy: { name: string } | null;
  submittedAt: string | null;
  confirmedBy: { name: string } | null;
  confirmedAt: string | null;
  deliveredBy: { name: string } | null;
  deliveredAt: string | null;
}
```

Renders only segments where the corresponding `*By` is non-null. Segments separated by ` · ` (space + U+00B7 + space). Time formatted as `HH:mm` (Swedish 24-hour convention; use `format(new Date(timestamp), 'HH:mm')` from `date-fns`).

**Visual:**

```jsx
<p className="text-xs text-muted-foreground flex flex-wrap gap-x-1 gap-y-1 mt-4">

  {deliveredBy && <span>· Levererad av {deliveredBy.name} {HH:mm}</span>}
</p>
```

- Typography: `text-xs text-muted-foreground`.
- No ARIA role needed — purely informational text in document flow.
- Rendered in Mode D (partial) and Mode E (full). Not rendered in Modes A, B, or C (no actor stamps yet visible in Phase 4 on those modes — though Mode B could show `Skickad av` as a future enhancement; it is not in scope).
