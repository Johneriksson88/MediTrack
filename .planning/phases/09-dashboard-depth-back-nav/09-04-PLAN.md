<objective>
Plan 04 — Gap closure (single gap from `09-VERIFICATION.md`).

Resolve `dashboard-wide-screen-whitespace`: at ≥1024px viewports both dashboard cards (especially `Läkemedel under tröskel`) render with excessive vertical whitespace inside their card frames. Structural layout (2-col at md+, stacked-mobile, no overflow) is correct; only the per-card sizing inside the grid cell is wrong.

Root cause (read from existing files):
1. `DashboardLowStockCard.tsx` data branch caps its scroll region at `max-h-80 overflow-y-auto` (line 113). When the orders card next to it is taller (two sections + multiple rows), the grid row stretches to the taller sibling's height, leaving up to ~280px of empty space below the capped low-stock card.
2. Neither card declares `h-full` or `flex flex-col`, so the shorter card cannot extend its visual frame to match the grid row height — the empty space is OUTSIDE the card body but INSIDE the grid cell, perceived as "the card frame is undersized for the available canvas".

Fix (FE-only, Tailwind className changes only — bounded per gap severity `minor`):
- Grid container: add `items-stretch` (force both grid items to fill the row height) ONLY. Container width cap (`max-w-5xl`) is unchanged because CONTEXT.md `<discretion>` line 141 advises against breakpoint-tier amendments above `md` on the grid container; widening the container at `lg+` would push against that discretion. The stretch-asymmetry fix below is the dominant lever closing the wide-screen whitespace symptom anyway — cards grow to fill their grid row, eliminating the "lower-half empty" perception independent of container width.
- DashboardLowStockCard: drop `max-h-80` from the data-branch CardContent; replace with `flex-1 overflow-y-auto` (grows with the grid cell, still scrolls if content exceeds the cell). Card declaration gains `h-full flex flex-col` so the Card itself fills the grid cell and the CardContent's `flex-1` has a height context. Add `data-testid="dashboard-low-stock-card-data"` on the Card and `data-testid="dashboard-low-stock-card-content"` on the CardContent so the test (see DashboardLowStockCard.test.tsx Task 5) can query them deterministically.
- DashboardOrdersCard: same `h-full flex flex-col` on the data-branch Card; data-branch CardContent gains `flex-1` (no need to drop a `max-h-*` because Plan 03 never added one). Empty-state and loading branches unchanged (those wrap in a centered Card with their own height). Add `data-testid="dashboard-orders-card-data"` on both data-branch Cards (nurse and pharmacist/admin) and `data-testid="dashboard-orders-card-content"` on the matching CardContents.
- DashboardOrdersCard test: add Test 10 (parameterized via `it.each` across nurse/apotekare/admin) asserting the new sizing classNames are present on the rendered Card + CardContent, queried via the new `data-testid` hooks so the test is deterministic regardless of shadcn Card primitive's class name details.
- DashboardLowStockCard test: add Test 6 mirroring Test 10's invariants — asserts the data-branch Card carries `h-full flex flex-col` and CardContent carries `flex-1` and does NOT carry `max-h-80`.

What is NOT touched (Hard Constraint 4): no other Phase 9 production file. No BE. No new requirements. No copy churn. No new D-NNN decision — this is an amendment-by-bounded-CSS of D-145's sizing, not a rejection of D-145 or D-146.

Purpose: close the single in-scope gap from `09-VERIFICATION.md` without disturbing the 7 production files the verifier already passed beyond the 3 production files explicitly listed in the gap's `surface:` field. The two test files in `files_modified` are not Phase 9 production files; they are extended in lockstep with the source changes (same convention Plan 03 already used).

Output: 3 edited production files + 2 extended test files. The harness re-capture of `docs/screenshots/sc04-360-dashboard.png` is EXPLICITLY DEFERRED to a post-phase chore per `09-CONTEXT.md <specifics>` lines 351–353 (which already classify the sc04 re-capture as "Not a Phase 9 commit in itself, but a follow-on in the same plan or as a wrap-up chore"). The new classes (`items-stretch`, `h-full flex flex-col`, `flex-1`) do NOT widen the 360px baseline: no `sm:` or base class touched on the container; cards stack via `grid-cols-1` baseline; `flex flex-col` + `h-full` on cards still collapse correctly because the grid row at `<md` is its content height, not a stretched row.
