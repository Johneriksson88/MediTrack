## Spacing Scale

Carry-forward from Phase 1 + 2 — no changes. All layout values multiples of 4px.

| Token | px | Tailwind class | Primary usage in Phase 3 |
|-------|----|----------------|--------------------------|
| 1 | 4px | `gap-1` / `p-1` | Icon-label gap in status pill, stepper button inner padding |
| 2 | 8px | `gap-2` / `p-2` | Footer summary bullets, stepper input ↔ button gap |
| 3 | 12px | `px-3` | Status pill horizontal padding (`px-3 py-1`) |
| 4 | 16px | `p-4` / `gap-4` | Compose-view header padding, card padding, sticky-footer outer padding, row gaps |
| 5 | 20px | `py-5` | Submit confirmation banner vertical padding |
| 6 | 24px | `p-6` / `gap-6` | Drafts-list page padding at md, section spacing |
| 8 | 32px | `p-8` | Page padding at lg+ |
| 12 | 48px | `py-12` | Empty-state vertical breathing room |

**Phase 3 additions / exceptions:**

- **Stepper buttons (`−` / `+`):** **44 × 44 px square buttons** at every breakpoint per Phase 1 touch-target floor — overrides default `Button size="icon"` (36 × 36 px). Use `h-11 w-11` explicitly. D-60.
- **Quantity input box (in stepper):** `w-16 h-11` (64 × 44 px). 11-row height matches the stepper buttons; 64 px width comfortably holds 4-digit quantities.
- **Sticky compose-view footer (mobile, `<md`):** outer wrapper `p-4`, then **bottom padding extended** with `pb-[calc(1rem+56px+env(safe-area-inset-bottom))]` to clear the 56 px bottom tab bar + iOS home indicator. Mirrors the Phase 2 Sheet-footer pattern verbatim — same geometry, different host element.
- **Sticky compose-view footer (≥md):** `p-4 border-t border-border`, no safe-area padding (no bottom tab bar at md+); sticks to the bottom of the main content scroll container, not the viewport, so it sits inside the sidebar layout.
- **`<OrderStatusPill>`:** `rounded-full px-3 py-1 text-xs font-semibold` (matches `<RoleBadge>` shape — `px-3` chosen over `px-2` because status labels run longer: `Bekräftad` / `Levererad`).
- **Compose-view line card (mobile):** `bg-card border border-border rounded-lg p-4 shadow-sm`, identical to Phase 2 `<MedicationCard>`. The right-edge trash button is `h-11 w-11` (44 × 44) per touch-target floor.
- **Table cell padding (compose-view table at ≥md):** `px-4 py-3` — carry-forward from Phase 2's `<MedicationTable>` so the two table surfaces feel identical.
- **Drafts-list table row padding:** `px-4 py-3` — same as Phase 2.
- **Submit confirmation banner (post-submit):** `mt-4 mx-4 sm:mx-0 rounded-lg border border-primary/20 bg-primary/10 px-4 py-3` — pale-blue tint, mirroring Phase 2's low-stock banner geometry but in accent (informational, not destructive).

### Accent Reserved For (Phase 3 additions to Phase 1 + 2 list)

1. **`Ny beställning` button** (drafts-list page, top-right desktop) — `Button variant="default"` (blue bg + white text).
2. **`Ny beställning` FAB** (mobile, drafts-list) — same blue, mirrors Phase 2 `<AddMedicationButton>` FAB geometry.
3. **`Skicka beställning` button** (compose-view sticky footer) — `Button variant="default"`, the primary CTA of the phase.
4. **`Lägg till läkemedel` button** (compose-view sticky footer + line-list empty-state action) — `Button variant="default"`. NB: same label as Phase 2 add-button, but here it triggers the **picker** Sheet, not the catalog create Sheet.
5. **Submit confirmation banner background tint** — `bg-primary/10` (pale blue) with `text-primary` body text. Informational, not warning. Mirrors the Phase 2 low-stock banner shape but in accent instead of destructive.
6. **`<OrderStatusPill status="skickad">`** background — `bg-primary/10 text-primary` (pale blue tint). This is the only status pill rendered in Phase 3 outside of `utkast`. See §`<OrderStatusPill>` color map.
7. **Focus rings** — `focus-visible:ring-2 focus-visible:ring-primary focus-visible:ring-offset-2` on stepper buttons, line trash icons, picker row buttons, and all other Phase 3 interactives.
8. **Active drafts-list row link / "Öppna" affordance** on focus / hover — `hover:bg-muted/50` (no accent on hover state per Phase 2 precedent; accent reserved for the focus ring).

**Accent must NOT appear on:** line cards (mobile), table rows (default bg), the picker Sheet header bg, line trash buttons (those are destructive), or the `Kasta` button.

### Destructive Reserved For (Phase 3 additions)

1. **`Kasta` button** (compose-view sticky footer, left-aligned) — `Button variant="destructive"`. D-67. The discard affordance.
2. **`Kasta detta utkast?` AlertDialog confirm button** — `AlertDialogAction` with destructive styling. D-67.
3. **Line trash icon button** (per-line removal on the compose view) — `Trash2` lucide icon, button is `ghost` variant but icon color is `text-destructive`. Touch target 44 × 44 px. Per-line removal is pessimistic-mutation (D-52) → reuses Phase 2's destructive-action color treatment.
4. **409 `order_locked` toast** — `toast.error(...)` with copy `Beställningen kan inte ändras efter att den skickats.` D-55, D-70.
5. **`<LowStockBadge>` in picker rows** — verbatim reuse of Phase 2's `<LowStockBadge>` component (bg-destructive + white text + AlertTriangle icon). D-61.

┌─────────────────────────────────────────────────────┐

- `<Input type="number" inputMode="numeric" min={1} step={1} className="h-11 w-16 rounded-none text-center text-sm font-semibold" />`
- `<Button variant="outline" size="icon" className="h-11 w-11 rounded-l-none border-l-0" aria-label="Öka antal">+</Button>`
- Visual unit: three controls share a single border outline; the input has no outline on its left/right edges to keep the stepper visually unified.

**Interaction (optimistic per D-52):**
1. Click `+` / `−`: optimistically update local state + TanStack Query cache via `useMutation.onMutate`.
2. **Debounced PATCH at 250 ms** after the last interaction (D-51, D-60) — `PATCH /api/orders/:id/lines/:lineId` with `{ quantity }`. Multiple rapid clicks coalesce.
3. Input typing: same 250 ms debounce. Also commit on `blur` if debounce hasn't fired yet.
4. **Long-press auto-repeat:** 250 ms initial delay, then 100 ms repeat interval. Released on `pointerup` / `pointercancel`. D-60.
5. **On `onError`:** rollback cache. `toast.error('Kunde inte spara — försök igen.')`. Input retains the user's last typed value so they can retry.
6. **On `order_locked` 409:** `toast.error('Beställningen kan inte ändras efter att den skickats.')` + invalidate `['order', id]` (switches the page to Mode B).
7. **Min enforcement:** `−` is disabled when `value === 1`. Typing `0` or below is silently coerced to `1` on blur with no toast (defensive — true `0` would fail submit validation per D-56). Negative numbers blocked at input (`min={1}`).

**Locked mode (`isLocked === true`):** All three controls render as a single `text-sm font-semibold` static span showing the quantity. No buttons. No input. Same width as the active stepper to avoid layout shift between Mode A and Mode B. Used in Mode B (`skickad`) line rendering.

**Accessibility:**
- `aria-label="Minska antal"` and `aria-label="Öka antal"` on the buttons.
- `aria-label="Antal"` on the input.
- Both buttons are reachable by keyboard tab; `Enter` / `Space` activate them.
- Screen reader announces the new value naturally after focus / blur (the `<input>` is the source of truth).
- `e.stopPropagation()` on stepper keydown to prevent any wrapping `<TableRow>` click handler from firing (though Mode A rows are not row-clickable — only Mode B locked rows would conflict, and there's no stepper there).

| Quantity PATCH 409 | Cache rollback + toast + invalidate `['order', id]` → page re-renders into Mode B |
| Add-line POST 409 | Re-open picker (if closed) + toast + invalidate `['order', id]` |
| Remove-line DELETE 409 | Re-enable trash button + toast + invalidate `['order', id]` |
| Submit POST 409 (raced — another tab won) | Toast + invalidate `['order', id]` → re-render Mode B |
| Discard DELETE 409 | Close dialog + toast + invalidate `['order', id]` → re-render Mode B (cannot delete a submitted order) |

## Responsive Breakpoint Contracts

| Breakpoint | Drafts list | Compose view body | Compose view actions |
|-----------|-------------|-------------------|----------------------|
| <768px (mobile) | Card list; FAB bottom-right | Card list (stacked) | Sticky footer fixed to viewport above bottom tab bar |
| ≥768px (md) | Table; top-right `Ny beställning` button | Table | Sticky footer inside content scroll container |
| ≥1024px (lg) | Same as md | Same as md | Same as md |
| ≥1440px (xl) | Same as md | Same as md | Same as md |

**No horizontal scroll at any breakpoint** (UX-01 obligation). Tables use `overflow-x-auto` wrapper. All flex children that must not overflow use `min-w-0`.

**Sheet:** `side="right"` on `md`+, `side="bottom"` on `<md`. Same approach as Phase 2 (`useMediaQuery('(min-width: 768px)')` or a Tailwind-based CSS toggle).

**Sticky footer z-index hierarchy (mobile):**
- Bottom tab bar: `z-50`
- Compose sticky footer: `z-40`
- Picker Sheet (when open): `z-50` (matches Radix Dialog default — overlays both)
- AlertDialog (when open): `z-50` (matches Radix AlertDialog default — overlays both)

The compose footer at `z-40` sits visually above the page content but **below** the bottom tab bar so the user can always reach Konto / Dashboard nav. Spatially, the footer's bottom padding (`pb-[calc(1rem+56px+...)]`) clears the tab bar so the action row sits above it without occlusion.

## Accessibility Floor

Additions to Phase 1 + 2 baseline (both carry forward unchanged).

| Requirement | Implementation |
|-------------|----------------|
| Stepper buttons keyboard | `tabIndex={0}` (default for `<button>`); `Enter`/`Space` activate; `aria-label="Öka antal"` / `Minska antal` |
| Stepper input | `aria-label="Antal"`; `inputMode="numeric"`; `min={1}`; `step={1}` |
| Trash button per line | `<Button aria-label="Ta bort rad">` containing only the `Trash2` icon (`aria-hidden="true"`) |
| Sticky footer landmark | `<footer aria-label="Åtgärder för beställning">` wrapping the sticky-footer content (helps screen readers identify it as a separate region from the line list above) |
| Submit-confirmation banner | `role="status"` (non-interrupting announce on Mode B mount) |
| Discard dialog focus | Cancel default-focused (shadcn default — do not override); `Escape` cancels |
| Picker Sheet title | `<SheetTitle>Lägg till läkemedel</SheetTitle>` always present — Radix Dialog requirement |
| Picker row | `<button>` with `min-h-[56px]` (exceeds 44 px touch target); accessible name = name + atc + form + stock concatenated naturally in the row text |
| Status pill | Plain `<span>` — informational only, not interactive, no `aria-label` needed (text is the label) |
| Drafts-list row click target | Entire `<TableRow>` is `cursor-pointer` + `aria-label="Öppna utkast skapat ..."` + `onClick` with `tabIndex={0}` + `onKeyDown` for `Enter`/`Space` |
| Drafts-list card click target | Whole-card `<button>` with `aria-label="Öppna utkast skapat ..."` |
| 409 toast | Sonner toasts have `role="status"` by default; explicit `aria-live="polite"` on the Toaster instance is the Phase 2 setup carry-forward |
| Long-press auto-repeat | Stops on `pointercancel` (e.g. scroll, system interrupt) to avoid runaway increment; respects `prefers-reduced-motion` only insofar as no animation is added |
| `prefers-reduced-motion` | All Sheet/dialog animations defer to Radix defaults; no Phase 3 motion is added beyond `Loader2 animate-spin` (always-on; spin is the affordance) and Sheet slide-in (Radix-controlled, respects reduced motion) |
| Focus trap | Sheet + AlertDialog focus traps are Radix-native; no override |
| 44 × 44 px touch targets | Stepper buttons (`h-11 w-11`), line trash button (`h-11 w-11`), picker rows (`min-h-[56px]`), drafts-list row chevron (parent row is the target — full row height ≥ 48 px via `py-3` padding + line height), FAB (`h-14 w-14`) |

## Open Questions Resolved

The orchestrator flagged these as potentially open after reading CONTEXT.md. All resolved in this UI-SPEC with sensible defaults; documented here so the checker can audit the reasoning.

| Question | Resolution | Rationale |
|----------|-----------|-----------|
| `<OrderStatusPill>` icon? | **No icon.** | Mirrors `<RoleBadge>` (no icon). Text + color is enough. Keeps pill compact in narrow table cells and the compose-view header. |
| `<OrderStatusPill>` color map? | utkast=slate / skickad=blue / bekraftad=amber / levererad=emerald (each `bg-N-100` + `text-N-700/800`). | Slate = neutral (passive draft); blue = "in transit" (matches `<RoleBadge role="apotekare">` lineage); amber = pending action (warning per Phase 2 below-threshold chip semantics); emerald = success terminal. All four pass WCAG AA. |
| Sticky-footer height + padding (mobile)? | Outer `p-4` + `pb-[calc(1rem+56px+env(safe-area-inset-bottom))]`. Main content gets matching `pb-[calc(56px+56px+env(safe-area-inset-bottom))]`. | Verbatim Phase 2 Sheet-footer pattern. Clears tab bar + iOS home indicator. |
| Status-pill column on drafts list? | **No.** | Phase 3 list is `?status=utkast` exclusively — the column would be redundant. Phase 4 introduces the multi-status history view and is the natural place for a pill column. |
| Empty-state icon (drafts list)? | `ClipboardList`. | Already in the nav for `Beställningar`. Keeps icon vocabulary consistent across the route. |
| Skeleton shapes (drafts list)? | Table: 5 × `h-12` `<Skeleton>` rows. Cards: 3 × `h-24` `<Skeleton>` cards. | Mirrors Phase 2 skeleton geometry. |
| Skeleton shapes (compose view)? | Header chrome renders immediately; line area = 3 × `h-16` cards on mobile / 3 × `h-12` rows in table on desktop. Footer hidden during loading. | Avoids flicker before status is known. |
| Focus on Sheet open? | **Search input autofocused.** | Picker presents N matching rows; the user's intent is to type-narrow. Focusing the input lets them filter immediately. First row gets focus on Tab. |
| Confirmation `<AlertDialog>` before Submit? | **No.** | Submit is irreversible (ORD-03) but low-cost — apotekare hasn't acted yet. Adding a confirm would friction the most-used action of the phase. Disabled-predicate + the full-verb button label (`Skicka beställning`) is the safety net. (Phase 4 will introduce confirmation dialogs for `Bekräfta` / `Leverera`, where the action is genuinely consequential — stock decrement.) |
