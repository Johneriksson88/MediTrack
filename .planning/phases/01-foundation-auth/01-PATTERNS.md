## Pattern Assignments

Since there are no in-repo analogs, each section names the **canonical pattern** the executor should implement, drawn from the locked decisions and the chosen frameworks. Executors should NOT copy code from external docs verbatim — use these as the shape, then write the code idiomatically.

### Pattern A — Prisma schema (D-01, AUTH-04, AUTH-07)

**Applies to:** `apps/api/prisma/schema.prisma`
**Source:** D-01 (sessions table), AUTH-04 (Role enum), AUTH-07 (one vårdenhet per user)

**Conventions to follow:**
- All models use `id String @id @default(cuid())` — stable across migrations, sortable, URL-safe.
- Timestamps: `createdAt DateTime @default(now())`, `updatedAt DateTime @updatedAt`. Both fields on every mutable model.
- Role as enum: `enum Role { apotekare sjukskoterska admin }` — snake-case in DB, matches `ROLES` constant in shared.
- Foreign keys: explicit relation name only when there are >1 relations between the same pair of models. Phase 1 doesn't need any.
- Indexes: index every FK used in a `where` clause (`@@index([careUnitId])` on User; `@@index([userId])` and `@@index([expiresAt])` on Session).
- Sessions table fields: `id String @id` (random opaque, generated in app code, NOT cuid), `userId`, `careUnitId` (denormalized snapshot — see D-16), `createdAt`, `expiresAt`, `lastSeenAt`. No `updatedAt`.

**Canonical shape:**
```prisma
enum Role {
  apotekare
  sjukskoterska
  admin
}

model CareUnit {
  id        String   @id @default(cuid())
  name      String
  createdAt DateTime @default(now())
  users     User[]
}

model User {
  id           String   @id @default(cuid())
  email        String   @unique
  name         String
  passwordHash String
  role         Role
  careUnitId   String
  careUnit     CareUnit @relation(fields: [careUnitId], references: [id])
  sessions     Session[]
  createdAt    DateTime @default(now())
  updatedAt    DateTime @updatedAt
  @@index([careUnitId])
}

model Session {
  id         String   @id        // opaque random, NOT cuid — set by app
  userId     String
  careUnitId String                // snapshot at session creation
  user       User     @relation(fields: [userId], references: [id], onDelete: Cascade)
  createdAt  DateTime @default(now())
  expiresAt  DateTime
  lastSeenAt DateTime @default(now())
  @@index([userId])
  @@index([expiresAt])
}
```

### Pattern B — Fastify app composition

**Applies to:** `apps/api/src/app.ts`, `apps/api/src/server.ts`
**Source:** Fastify v4+ idiomatic bootstrap, D-19 (error handler), D-02 (no CORS in dev)

**Conventions:**
- Export `buildApp(): FastifyInstance` from `app.ts`. `server.ts` is a thin wrapper that calls `buildApp()` and `app.listen({ port, host: '0.0.0.0' })`. This split is **mandatory** so tests can build the app without binding a port.
- Use `app.setErrorHandler(...)` once, in a plugin (`plugins/errorHandler.ts`), to translate thrown errors into the canonical envelope.
- Register plugins in this order: env validation -> cookies -> errorHandler -> routes.
- No CORS plugin needed (D-02). If added, justify in commit.
- Use the JSON Schema route option (`schema: { body: ..., response: ... }`) wired to Zod schemas via `fastify-type-provider-zod` (or hand-converted). The schemas come from `@meditrack/shared`.

### Pattern C — Fastify preHandler factory (auth + RBAC)

**Applies to:** `apps/api/src/auth/requireSession.ts`, `apps/api/src/auth/requirePermission.ts`
**Source:** D-15, D-19

**Conventions:**
- preHandler signature: `async (req: FastifyRequest, reply: FastifyReply): Promise<void>`. Throw via `reply.code(...).send(envelope)` and `return` — don't `throw` non-Fastify errors from preHandlers.
- `requireSession` reads the session cookie, looks up the session row, checks expiry, touches `lastSeenAt` and bumps `expiresAt` (sliding 7d, 30d cap — D-03), decorates `req.user = { id, role, careUnitId, name, email }`, and continues. On any failure: `reply.code(401).send({ error: { code: 'unauthenticated', message: 'Du måste logga in.' } })`.
- `requirePermission(action: ActionKey)` returns a preHandler that assumes `req.user` is set (it always runs after `requireSession` in the chain). If `!PERMISSIONS[action].includes(req.user.role)`: `reply.code(403).send({ error: { code: 'forbidden', message: 'Du saknar behörighet att utföra denna åtgärd.' } })`.
- Route registration pattern:
  ```ts
  app.get('/api/admin/ping', { preHandler: [requireSession, requirePermission('admin:ping')] }, handler);
  ```
- **Order matters:** session check first, permission check second. Never reorder.

### Pattern D — Service-layer tenant scoping (D-16)

**Applies to:** `apps/api/src/services/*.ts` (all of them — sets the precedent on `auth.service.ts` and `user.service.ts`)
**Source:** D-16

**Conventions:**
- Every service function takes `careUnitId: string` as the **first argument**. No exceptions, even when the function obviously doesn't need it (Phase 1 sets the discipline).
- Every Prisma query inside a service includes `where: { careUnitId, ... }`. Never trust the route handler to filter.
- Route handlers MUST NOT call `prisma.*` directly. They call services. Enforce in code review; Phase 1 commits should never show a handler importing `db/client`.
- Exception (Phase 1 only): `auth.service.ts#login` does not receive a `careUnitId` because the user hasn't been identified yet. It reads `careUnitId` from the matched User row and returns it in the session payload. Document this exception in a comment.

### Pattern E — Canonical error envelope (D-19)

**Applies to:** `packages/shared/src/contracts/error.ts`, `apps/api/src/plugins/errorHandler.ts`, every route that returns an error
**Source:** D-19

**Canonical shape:**
```ts
// packages/shared/src/contracts/error.ts
import { z } from 'zod';

export const errorEnvelope = z.object({
  error: z.object({
    code: z.string(),                  // 'invalid_credentials', 'forbidden', 'unauthenticated', 'validation_failed', ...
    message: z.string(),               // Swedish, user-displayable
    details: z.unknown().optional(),
  }),
});
export type ErrorEnvelope = z.infer<typeof errorEnvelope>;
```

- Every non-2xx response from the API matches this schema. The error handler plugin enforces it for thrown errors; explicit `reply.code(4xx).send(envelope)` calls do so manually.
- The `code` is a short stable English string. The `message` is Swedish. The FE may key error UX off `code`, never off `message`.

### Pattern F — Zod schema as shared contract (D-08)

**Applies to:** every file in `packages/shared/src/contracts/`
**Source:** D-08

**Conventions:**
- Each contract file exports both the schema and the inferred type, in lowercase + PascalCase:
  ```ts
  // packages/shared/src/contracts/login.ts
  import { z } from 'zod';

  export const loginRequest = z.object({
    email: z.string().email(),
    password: z.string().min(1),
  });
  export type LoginRequest = z.infer<typeof loginRequest>;

  export const loginResponse = z.object({
    user: z.object({ /* ... */ }),
  });
  export type LoginResponse = z.infer<typeof loginResponse>;
  ```
- Fastify route uses `loginRequest` as the body schema. React form uses `zodResolver(loginRequest)`. **Same import path** (`@meditrack/shared`).
- Never duplicate a schema. If two endpoints share a shape, extract it.
- Schemas never reference Prisma types — Prisma is a backend-only detail (D-07).

### Pattern G — Password hashing (D-05)

**Applies to:** `apps/api/src/auth/password.ts`
**Source:** D-05

**Conventions:**
- Use `argon2` npm package, `argon2id` variant, OWASP 2025 defaults (memoryCost ~19456 KiB, timeCost 2, parallelism 1 — confirm against `argon2.defaults` at install).
- Export exactly two functions: `hashPassword(plain: string): Promise<string>` and `verifyPassword(hash: string, plain: string): Promise<boolean>`.
- Never log the plain password or hash. Hash never leaves the server.
- Fallback note: if argon2 native build fails on Windows during local dev, swap to `bcrypt`. Document in README; the function signature stays the same.

### Pattern H — Session lifecycle (D-01, D-03)

**Applies to:** `apps/api/src/auth/session.ts`
**Source:** D-01, D-03

**Conventions:**
- Session ID: 32 bytes from `crypto.randomBytes(32)`, base64url-encoded (43 chars, no padding).
- Exported functions: `createSession(userId, careUnitId): Promise<{ id, expiresAt }>`, `findSessionById(id): Promise<Session | null>`, `touchSession(id): Promise<Session>` (bumps `lastSeenAt = now`, `expiresAt = min(now+7d, createdAt+30d)`), `destroySession(id): Promise<void>`.
- The 30-day cap is computed against `session.createdAt`, NOT against the user's first-ever login.
- All four functions take the Prisma client implicitly via `db/client.ts` singleton.

### Pattern J — Permission map (D-15, D-17)

**Applies to:** `apps/api/src/auth/permissions.ts`, `packages/shared/src/contracts/permissions.ts`
**Source:** D-15, D-17, D-18

**Conventions:**
- `ActionKey` is a `z.ZodLiteral` union or a TS literal union — exported from `@meditrack/shared/contracts/permissions.ts` so both FE and BE import the **exact same set of strings**.
- For Phase 1, the only action is `'admin:ping'` (D-18). Later phases add `'medication:create'`, `'order:confirm'`, `'order:deliver'`, `'audit:read'`, etc.
- The map lives in `apps/api/src/auth/permissions.ts`:
  ```ts

  import type { Role } from '@prisma/client';

  };
  ```

### Pattern K — TanStack Query + React Router v7 split (D-12, D-13)

**Applies to:** `apps/web/src/router.tsx`, `apps/web/src/auth/AuthGate.tsx`, `apps/web/src/lib/queryClient.ts`
**Source:** D-12, D-13

**Conventions:**
- Router defines routes (URL -> component). It does NOT fetch data. Loaders are not used in Phase 1.
- TanStack Query owns all data: `useQuery(['me'])` is the source of truth for `useAuth`. Mutations (`useMutation`) call API and invalidate `['me']` (or other keys) on success.
- `AuthGate`:
  ```tsx
  function AuthGate({ children }: PropsWithChildren) {
    const location = useLocation();
    const { data, isLoading, isError, error } = useQuery({ queryKey: ['me'], queryFn: fetchMe, retry: false });
    if (isLoading) return <AuthSkeleton />;
    if (isError && isUnauthenticated(error)) return <Navigate to="/login" state={{ from: location.pathname }} replace />;
    if (isError) throw error;          // Router error boundary handles unexpected
    return <>{children}</>;
  }
  ```
- QueryClient defaults: `retry: 1`, `refetchOnWindowFocus: false` (interview demo doesn't need it), `staleTime: 30_000`.

### Pattern L — `useAuth` + `<Can>` (D-17)

**Applies to:** `apps/web/src/auth/useAuth.ts`, `apps/web/src/auth/useCan.ts`, `apps/web/src/auth/Can.tsx`
**Source:** D-17

**Canonical shapes:**
```ts
// useAuth.ts
export function useAuth() {
  const { data, isLoading } = useQuery({ queryKey: ['me'], queryFn: fetchMe, retry: false });
  const can = useCallback((action: ActionKey) => data?.permissions.includes(action) ?? false, [data]);
  return { user: data ?? null, isLoading, can };
}

// useCan.ts
export function useCan(action: ActionKey): boolean {
  const { can } = useAuth();
  return can(action);
}

// Can.tsx
export function Can({ action, children }: { action: ActionKey; children: ReactNode }) {
  return useCan(action) ? <>{children}</> : null;
}
```
- `<Can>` returns `null` (not a fallback) for non-permitted users. The UI-SPEC §403 / `<Can>` Gate Pattern handles the muted note pattern at the route level, not inside `<Can>`.

### Shared #1 — All API responses match `errorEnvelope` (errors) or a documented success schema

**Source:** D-19

**Why first-class:** Phases 2-7 all rely on this. If Phase 1 ships an inconsistent envelope, every later phase has to fight it.

**Why first-class:** Sets the multi-tenancy discipline. Phase 1's `auth.service.ts` and `user.service.ts` are the canonical examples future phases copy from.

### Shared #3 — Zod schemas in `packages/shared` are the only cross-package types

**Source:** D-08
**Apply to:** every BE-FE wire shape from Phase 1 forward
**Why first-class:** Prisma types stay backend-only (D-07). If a type needs to cross the wire, it goes through a Zod schema in `@meditrack/shared`.

### Shared #4 — `ActionKey` permission strings live in shared; map lives on the server

**Source:** D-15, D-17, D-18
**Apply to:** Every permission added in Phase 2+
**Why first-class:** Two sources of truth for permissions is the most common drift bug. Phase 1 locks one source.

### Shared #5 — Swedish UI strings, English code identifiers

**Source:** CONTEXT.md §Specific Ideas, UI-SPEC §Copy
**Apply to:** every component, every error message, every constant
**Why first-class:** UI-SPEC has the exact strings already. The discipline is: a string the reviewer sees is Swedish; an identifier they read in code is English.

### Shared #6 — Use `focus-visible` + the locked color ring on every interactive element

**Source:** UI-SPEC §Accessibility Floor, §Focus Ring Specification
**Apply to:** every button, link, input, nav item
**Why first-class:** Hard accessibility floor; UX-01 is graded.

### Shared #7 — Touch targets ≥ 44×44 px, bottom tab bar respects safe-area inset

**Source:** UI-SPEC §Spacing Scale
**Apply to:** every interactive element in mobile chrome and stub pages
**Why first-class:** Mobile-first is explicit in UX-01 and reviewer rubric.

### Shared #8 — TanStack Query owns data; React Router owns URL

**Source:** D-12
**Apply to:** every route and every fetch from Phase 1 forward
**Why first-class:** Mixing loaders + Query later is messy. Phase 1 locks the split.
