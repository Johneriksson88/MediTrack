Zod contracts (packages/shared/src/contracts/):
- errorEnvelope: { error: { code: string, message: string, details?: unknown } }              (Pattern E)
- loginRequest:  { email: string (email), password: string (min 1) }                          (Pattern F, M)
- loginResponse: { user: { id, email, name, role, careUnit: { id, name } } }                   (D-18)
- meResponse:    { id, email, name, role, careUnit: { id, name }, permissions: ActionKey[] }  (D-18)
- ActionKey:     TS literal union — Phase 1 starts empty: `type ActionKey = never` placeholder; Plan 03 adds 'admin:ping' and the union widens

Fastify auth chain:
- requireSession(req, reply):                                                                  (Pattern C)
    1. read cookie SESSION_COOKIE (signed via @fastify/cookie)
    2. findSessionById; 401 if missing / expired
    3. touchSession (sliding 7d, 30d cap)
    4. decorate req.user = { id, role, careUnitId, name, email }
- routes/auth.ts:  POST /api/auth/login, DELETE /api/auth/session
- routes/me.ts:    GET /api/me (preHandler: [requireSession])
- routes/healthz.ts: GET /healthz (200 "ok", no auth)

Service layer (Pattern D, D-16):
- authService.login(email, password): { user, session: { id, expiresAt } } | throws InvalidCredentials
- authService.logout(sessionId): void
- userService.getMeForSession(careUnitId, sessionId): MeResponse
- NOTE: authService.login does NOT take careUnitId first — it's the one documented exception (Pattern D).

Frontend (Pattern K, L, M):
- queryClient: retry:1, refetchOnWindowFocus:false, staleTime:30_000
- fetchJson<T>(path, init?): Promise<T>  — credentials:'include', throws ApiError on non-2xx
- AuthGate({ children }): renders <AuthSkeleton/> while loading; <Navigate to="/login"/> on 401; throws on other errors
- useLogin(): useMutation<LoginResponse, ApiError, LoginRequest>
- LoginForm: react-hook-form + zodResolver(loginRequest), Alert variant="destructive" on invalid_credentials
