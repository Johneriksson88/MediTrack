<objective>
Slice A — Dashboard Low-Stock Banner end-to-end.

Replace the Phase 1 `<EmptyStateCard heading="Dashboard"/>` stub on `/dashboard` with a card that enumerates every `CareUnitMedication` in the user's `vårdenhet` whose `currentStock < lowStockThreshold`. Sorted server-side by urgency ratio. Backed by a new dedicated endpoint `GET /api/dashboard/low-stock`, owning its own cache key `['dashboard', 'low-stock']`. Auto-refreshes via three layers (TanStack invalidation on mutations + `refetchOnWindowFocus` + 30-second poll, per D-119). Empty state is celebratory (`Alla läkemedel är över tröskel.`).

Purpose: Satisfies REQ-IDs NTF-01 (full enumeration) and NTF-02 (auto-refresh on stock-changing mutation). No LLM dependency — demoable on a fresh `docker compose up` regardless of `ANTHROPIC_API_KEY`. This slice ships first because it is independently demoable and exercises Phase 4's invalidation infrastructure with the new sibling cache key (D-119).

Output: 9 new files + 4 edited files. One new GET endpoint, one new TanStack query hook, one new card component, two integration-test files.
</objective>
