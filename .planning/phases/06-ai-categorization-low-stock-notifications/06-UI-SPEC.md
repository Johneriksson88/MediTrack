## Components & States

### 1. DashboardLowStockCard (`apps/web/src/routes/dashboard/DashboardLowStockCard.tsx`)

Replaces the Phase 1 `<EmptyStateCard heading="Dashboard" />` stub on `DashboardPage.tsx`.

**Structure:**

```
<Card className="w-full">
  <CardHeader>
    <CardTitle>Läkemedel under tröskel</CardTitle>           ← text-xl font-semibold
    <CardDescription>{total} läkemedel</CardDescription>     ← text-sm text-muted-foreground
  </CardHeader>
  <CardContent className="max-h-80 overflow-y-auto">        ← scrollable when >5 rows
    {rows.map(row => <LowStockRow ... />)}
  </CardContent>
</Card>
```

**LowStockRow layout:**

```
<div className="flex items-center justify-between py-2 min-h-[44px]">
  <span className="text-sm font-normal">{row.name}</span>
  <div className="flex items-center gap-2">
    <span className="text-xs text-muted-foreground">
      {row.currentStock} / {row.lowStockThreshold}
    </span>
    <LowStockBadge />                         ← reused from Phase 2
  </div>
</div>
```

Between rows: `<hr className="border-border" />` (no `my-*` margin — spacing comes from row `py-2`).

**Sort order:** `currentStock / lowStockThreshold` ratio ASC (most urgent first). This sort is
applied server-side in `dashboard.service.ts` — the FE renders rows in received order.

**States:**

| State | Visual |
|-------|--------|
| Loading | `<Skeleton className="h-10 w-full" />` repeated 3× inside CardContent. No spinner text. |
| Loaded — empty (`total === 0`) | Celebratory `<EmptyStateCard>` reuse: `CheckCircle2` (emerald-600) + `Alla läkemedel är över tröskel.` heading + `text-sm text-muted-foreground` body `Alla läkemedel i din vårdenhet är över lagertröskeln.` |
| Loaded — non-empty | Full row enumeration per `rows` array |
| Error | `<Alert variant="destructive">` with `<AlertDescription>Kunde inte hämta lagernivåer — försök igen om en stund.</AlertDescription>` inside CardContent. No retry button (the 30-second poll handles it). |

**Refresh indicators:** No visible spinner on background refetches. TanStack Query's `isFetching`
state is not surfaced — the data refreshes silently. The loading skeleton only shows on the
initial mount (when `isLoading` is true, not `isFetching`).

**Mobile (< 768px):** Card occupies full content width (`w-full`). CardHeader and CardContent
inherit page padding (`p-4`). The inner scroll container (`max-h-80`) still applies.

**Tablet/desktop (≥ 768px):** Card uses `max-w-2xl` centered within the page's `max-w-7xl mx-auto`
container — wide enough to show 3 columns of data without horizontal scroll.

### 2. AiSuggestionChip (`apps/web/src/components/AiSuggestionChip.tsx`)

Read-only display component for the AI suggestion result. Rendered inside `MedicationSheet`.

**When no suggestion has been fetched:** Renders nothing — zero height, zero layout space. The
"Hämta AI-förslag" button is the only visible element in the AI row until a suggestion arrives.

**When a suggestion is available:**

```
<div className="flex items-center gap-2 bg-slate-50 border border-border rounded-md px-3 py-2">
  <span className="text-xs text-muted-foreground font-semibold">Förslag:</span>
  <span className="text-sm text-foreground">{THERAPEUTIC_CLASS_LABELS[therapeuticClass]}</span>
  <ConfidenceBadge confidence={confidence} />
</div>
```

Props: `{ therapeuticClass: TherapeuticClass, confidence: 'hog' | 'medel' | 'lag', onApply: () => void }`.

The `onApply` callback is triggered by the "Använd förslag" button, which renders **below** the
chip row (not inside it):

```
<Button variant="outline" size="sm" onClick={onApply} className="mt-1">
  Använd förslag
</Button>
```

`size="sm"` applies `h-9` (36px). This is an acceptable deviation from the ≥44 px rule because
the button is a secondary affordance inside an already-opened Sheet — interaction is deliberate,
not a primary touch target. Document this deviation in-code with a comment:
`{/* h-9 intentional: secondary affordance in Sheet; primary touch target is the Sheet itself */}`

## Interaction Contracts

### AI Suggestion Flow (full state machine)

```
[idle: no suggestion]
  → user clicks "Hämta AI-förslag"
  → button enters isFetching (Loader2 + "Hämtar förslag…", button disabled)
  → on success:
      → AiSuggestionChip appears (chip + ConfidenceBadge + "Använd förslag" button)
      → "Hämta AI-förslag" button returns to idle (user can fetch again)
  → on timeout (504):
      → toast.error("AI-förslaget tog för lång tid — försök igen.")
      → button returns to idle
  → on error (other):
      → toast.error("Kunde inte hämta förslag — försök igen.")
      → button returns to idle

[suggestion shown]
  → user clicks "Använd förslag":
      → "Slutgiltig klass" combobox updates to suggested TherapeuticClass value
      → chip remains visible (user can see what they accepted)
      → user can edit the combobox after applying (override flow)
  → user ignores suggestion:
      → "Slutgiltig klass" combobox stays at its current value (may be null)
  → user fetches again:
      → isFetching spinner again; new chip replaces old chip on success
```

### Dashboard Banner Refresh Flow

```
[tab open, dashboard visible]
  → initial load: skeleton → data appears
  → every 30 seconds: silent background refetch (no skeleton; data updates in-place)
  → user delivers an order (useDeliverOrder.onSuccess):
      → ['dashboard', 'low-stock'] query invalidated
      → immediate silent refetch (row count may decrease for the delivered med's row)
  → user Alt-tabs away then returns:
      → refetchOnWindowFocus fires immediately on window focus
      → silent refetch (no skeleton)
  → user creates/edits/deletes a medication (useMedicationMutations):
      → ['dashboard', 'low-stock'] query invalidated
      → immediate silent refetch (threshold change may flip a row's under-threshold status)
```

### Terapeutisk Klass Filter Interaction

```
[user on /lakemedel, no class filter active]
  → clicks "Terapeutisk klass" combobox trigger ("Alla klasser")
  → dropdown opens with 14 options + CommandInput search
  → user types "nerv" → list filters to "N — Nervsystemet"
  → user selects "N — Nervsystemet"
  → dropdown closes; trigger shows "Nervsystemet"; URL updates to ?class=N
  → medication list refetches with class=N filter applied
  → user clicks X on trigger → class param cleared; list refetches without class filter
```
