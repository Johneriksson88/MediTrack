**Critical detail for the planner:** which nav is rendered depends on viewport — `Sidebar` is `hidden md:flex` (renders only ≥ 768 px), `BottomTabBar` is `md:hidden` (renders only < 768 px). The SC#4 script checks all four viewports (360, 768, 1024, 1440), so the `data-test="primary-nav"` attribute MUST appear on **BOTH** nav elements (or on a wrapping container, but the simpler path is both `<nav>` elements). Otherwise:
- At 360 px: only `BottomTabBar` is in the DOM → if attribute is only on `Sidebar`, assertion fails.
- At 768+ px: only `Sidebar` is in the DOM → if attribute is only on `BottomTabBar`, assertion fails.

**Add to `Sidebar.tsx` line 28** (single attribute, no other changes):
```tsx
<nav className="flex flex-col" aria-label="Primary" data-test="primary-nav">
```

**Add to `BottomTabBar.tsx` line 25** (single attribute, no other changes):
```tsx
<nav
  aria-label="Primary"
  data-test="primary-nav"
  className="md:hidden fixed bottom-0 inset-x-0 h-14 bg-[#F1F5F9] border-t border-[#E2E8F0] flex pb-[env(safe-area-inset-bottom)]"
>
```

**Convention to establish (first-use):** plain HTML attribute `data-test`, no shared utility, no constants file. Direct string literal because (a) only used once per location, (b) cross-language searchability (grep-friendly), (c) zero runtime cost. If Phase 7 v2 grows a functional E2E suite, factor into `apps/web/src/test-locators.ts`; not now.
